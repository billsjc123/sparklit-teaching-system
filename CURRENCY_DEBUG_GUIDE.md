# 币种显示问题诊断指南

## 问题描述
更新学生费率时，显示的币种符号不正确（例如：更新人民币费率，但显示为 HK$200.00/节）

## 问题原因分析

### 最可能的原因
**在编辑学生信息时，只修改了"每节课费率"的数字，但没有同时修改旁边的"币种下拉选择器"。**

在编辑对话框中：
- 左侧输入框：费率数字（如 200）
- 右侧下拉框：币种选择（人民币 ¥ / 港币 HK$）

**这两个字段是独立的**，需要分别设置！

### 其他可能原因
1. 浏览器缓存了旧数据
2. 页面没有刷新
3. LocalStorage 数据损坏

## 解决方案

### 方案一：正确修改学生币种（推荐）

1. 访问系统：http://120.76.158.63
2. 进入"学生管理"页面
3. 点击要修改的学生右侧的"编辑"按钮（铅笔图标）
4. 在弹出的编辑对话框中：
   - **修改费率数字**（如改为 200）
   - **同时修改右侧的币种下拉框**：
     - 如果要设置为人民币，选择"人民币 (¥)"
     - 如果要设置为港币，选择"港币 (HK$)"
5. 点击"更新"按钮
6. 确认列表中显示的币种符号正确

### 方案二：使用调试工具检查数据

1. 访问调试页面：http://120.76.158.63/debug-currency.html
2. 查看学生数据表格：
   - **每节课费率** 列：显示带币种符号的费率
   - **币种** 列：显示币种名称（人民币/港币）
   - **币种代码** 列：显示存储的原始值（CNY/HKD）
3. 检查"币种代码"列的值：
   - 如果显示 `CNY` 但费率显示为 `HK$`，说明前端有显示逻辑错误
   - 如果显示 `HKD` 但你想要人民币，说明需要重新编辑并修改币种选择器

### 方案三：清除缓存重新测试

如果怀疑是缓存问题：

1. **清除浏览器缓存**：
   - Chrome/Edge：按 `Ctrl + Shift + Delete`，选择"缓存的图片和文件"
   - 或者使用无痕模式访问（Ctrl + Shift + N）

2. **清除 LocalStorage**：
   - 访问 http://120.76.158.63/debug-currency.html
   - 点击"🗑️ 清除 LocalStorage"按钮
   - ⚠️ **警告**：这会删除所有本地数据！

3. **强制刷新页面**：
   - 按 `Ctrl + F5` 或 `Ctrl + Shift + R`

## 验证步骤

修改后，请验证以下内容：

1. **学生管理页面**：
   - "每节课费率"列显示正确的币种符号
   - "币种"列显示正确的币种标签（蓝色=人民币，紫色=港币）

2. **课程安排页面**：
   - 课程列表中的费用显示正确
   - 课程详情对话框中的费用显示正确

## 代码逻辑说明

### 币种符号显示逻辑

```typescript
// 在 StudentsPage.tsx 第 269 行
{student.currency === 'CNY' ? '¥' : 'HK$'}{student.ratePerClass.toFixed(2)}/节
```

- 如果 `student.currency === 'CNY'`，显示 `¥`
- 否则显示 `HK$`

### 币种选择器

```tsx
<select
  id="currency"
  value={formData.currency}
  onChange={e => setFormData({ 
    ...formData, 
    currency: e.target.value as 'CNY' | 'HKD'
  })}
>
  <option value="CNY">人民币 (¥)</option>
  <option value="HKD">港币 (HK$)</option>
</select>
```
