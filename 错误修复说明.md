# 错误修复说明

## 问题描述

### 浏览器控制台错误
```
Uncaught Error: A <Select.Item /> must have a value prop that is not an empty string. 
This is because the Select value can be set to an empty string to clear the selection and show the placeholder.
```

**错误位置：** `SchedulesPage.tsx` - 教师筛选器

**错误原因：** Radix UI 的 `Select.Item` 组件不允许使用空字符串 `""` 作为 value 值

## 修复方案

### 原代码（错误）
```tsx
const [selectedTeacherId, setSelectedTeacherId] = useState<string>('');

// ...

<SelectItem value="">所有教师</SelectItem>
```

### 修复后代码
```tsx
const [selectedTeacherId, setSelectedTeacherId] = useState<string>('all');

// ...

<SelectItem value="all">所有教师</SelectItem>
```

## 具体修改

### 1. 修改初始状态
```tsx
// 从空字符串改为 'all'
const [selectedTeacherId, setSelectedTeacherId] = useState<string>('all');
```

### 2. 修改筛选逻辑
```tsx
const displayedSchedules = useMemo(() => {
  // 增加 'all' 的判断
  if (!selectedTeacherId || selectedTeacherId === 'all') return state.schedules;
  return state.schedules.filter(s => s.teacherId === selectedTeacherId);
}, [selectedTeacherId, state.schedules]);
```

### 3. 修改 SelectItem
```tsx
<SelectContent>
  <SelectItem value="all">所有教师</SelectItem>  {/* 改为 'all' */}
  {state.teachers.map(teacher => (
    <SelectItem key={teacher.id} value={teacher.id}>
      {teacher.name}
    </SelectItem>
  ))}
</SelectContent>
```

### 4. 修改条件显示
```tsx
// 排除 'all' 不显示课程数量
{selectedTeacherId && selectedTeacherId !== 'all' && (
  <span className="text-sm text-gray-600">
    共 {displayedSchedules.length} 节课程
  </span>
)}
```

### 5. 修改日期点击处理
```tsx
setFormData({
  ...formData,
  // 只有选中具体教师时才预填
  teacherId: selectedTeacherId !== 'all' ? selectedTeacherId : '',
  startTime: format(startTime, "yyyy-MM-dd'T'HH:mm"),
  endTime: format(endTime, "yyyy-MM-dd'T'HH:mm"),

```

### 6. 修改列表空状态提示
```tsx
{selectedTeacherId !== 'all' ? '该教师暂无课程安排' : '暂无课程安排，点击右上角添加课程'}
```

## 技术要点

### Radix UI Select 组件规则
- ✅ `value` 可以是任意非空字符串
- ❌ `value` 不能是空字符串 `""`
- ✅ 使用特殊值如 `"all"`, `"none"`, `"default"` 等
- ✅ 清空选择应该用特定值，不是空字符串

### 最佳实践
1. 使用语义化的特殊值（如 `"all"`, `"none"`）
2. 在状态初始化时使用特殊值
3. 在条件判断中包含特殊值的处理
4. 确保所有使用该状态的地方都正确处理特殊值

## 经验总结

### Radix UI 组件使用注意事项

1. **Select 组件**
   - 不能使用空字符串作为 value
   - 清空选择应使用特殊值
   - placeholder 仅在无选中值时显示

2. **其他可能的陷阱**
   - RadioGroup 也有类似限制
   - Toggle 组件的 value 不能为 undefined
   - Checkbox 的 checked 必须是 boolean

3. **调试技巧**
   - 查看浏览器控制台的详细错误堆栈
   - 检查 Radix UI 官方文档的约束条件
   - 使用 TypeScript 类型提示避免错误值
